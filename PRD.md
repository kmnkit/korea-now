# Product Requirements Document (PRD)
# Korea Now - 韓国トレンドスポット共有アプリ

## 📋 プロダクト概要

**プロダクト名**: Korea Now（コリアナウ）

**コンセプト**: 日本在住の韓国人や韓国に興味がある人が、最新の韓国トレンドスポット・飲食店情報を簡単に見つけて共有できるWebアプリケーション

**タグライン**: 「今の韓国」が、ここにある

---

## 🎯 背景と解決する課題

### 課題
- 日本在住の韓国人が、日本人に韓国旅行のおすすめを聞かれることが多い
- インドア派や流行に詳しくない人は、最新情報をすぐに答えられない
- 既存の観光情報サイトは古い情報や定番スポットばかり
- リアルタイムの「今」流行っている場所が分かりにくい

### 解決策
ユーザー同士がリアルタイムで韓国の最新トレンドスポットを投稿・共有できるプラットフォームを提供し、誰でも「今」の韓国のおすすめを即座に答えられるようにする。

---

## 👥 ターゲットユーザー

### プライマリユーザー
1. **日本在住の韓国人**
   - 韓国旅行のおすすめを聞かれることが多い
   - 最新トレンドを把握しておきたい
   - 母国の情報を手軽にチェックしたい

2. **韓国旅行予定者（日本人）**
   - 定番ではなく「今」流行っている場所に行きたい
   - 現地の人のリアルな声を聞きたい
   - SNSで映える場所を探している

### セカンダリユーザー
3. **韓国カルチャー好き**
   - K-POP、韓国ドラマ、韓国グルメなどに興味がある
   - 韓国の最新トレンドを常にチェックしたい

## 🎨 主要機能

### MVP (Minimum Viable Product)

#### 1. スポット投稿機能
- **カテゴリ選択**
  - カフェ
  - レストラン
  - 観光スポット
  - ショッピング
  - ナイトライフ
  - その他

- **必須情報**
  - スポット名
  - カテゴリ
  - エリア（ソウル、釜山、済州島など）
  - 簡単な説明（200文字以内）
  - 写真（最大3枚）

- **任意情報**
  - 住所
  - 営業時間
  - 予算感
  - タグ（#カップル #一人旅 #インスタ映え など）
  - Googleマップリンク

#### 2. スポット閲覧機能
- **フィルタリング**
  - カテゴリ別
  - エリア別
  - タグ別

- **ソート機能**
  - 新着順
  - 人気順（いいね数）
  - トレンド順（最近のいいね増加率）

- **詳細表示**
  - スポット情報
  - 投稿者プロフィール（簡易）
  - いいね数
  - コメント
  - 地図表示

#### 3. リアクション機能
- **いいね機能**
  - ワンクリックで気に入ったスポットを保存
  - いいね数でトレンド判定

- **コメント機能**
  - 簡単な感想や追加情報を投稿
  - 最大200文字

- **ブックマーク機能**
  - 後で見返したいスポットを保存
  - マイページから一覧表示

#### 4. ユーザー認証
- **サインアップ/ログイン**
  - メールアドレス + パスワード
  - Googleアカウント連携
  - （将来的に）LINE、Kakao Talk連携

- **プロフィール**
  - ニックネーム
  - アイコン画像（任意）
  - 自己紹介（100文字以内、任意）

#### 5. コンテンツモデレーション機能
- **投稿前AI自動チェック**
  - 画像：不適切な画像の検出（暴力、アダルトコンテンツなど）
  - テキスト：スパム、誹謗中傷、差別的表現の検出

- **ユーザー通報機能**
  - 不適切な投稿を通報
  - 通報理由の選択（スパム、不適切なコンテンツ、誤情報など）

- **管理者レビュー**
  - 通報された投稿を管理者が確認
  - 違反コンテンツの削除・ユーザーへの警告

#### 6. 検索機能
- **キーワード検索**
  - スポット名、説明文から検索
  - エリア名、タグからも検索可能

## 🛠 技術スタック

### フロントエンド
- **フレームワーク**: React + Next.js
- **スタイリング**: Tailwind CSS
- **状態管理**: React Context API / Redux Toolkit
- **地図**: Google Maps API / Kakao Maps API
- **画像**: Cloudinary / AWS S3

### バックエンド
- **フレームワーク**: Node.js + Express / Next.js API Routes
- **データベース**: PostgreSQL (Supabase)
- **認証**: NextAuth.js / Supabase Auth
- **ファイルストレージ**: Supabase Storage / AWS S3

### AI/モデレーション
- **画像モデレーション**: AWS Rekognition / Google Cloud Vision API
- **テキストモデレーション**: OpenAI Moderation API / Perspective API

### インフラ
- **ホスティング**: Vercel / Netlify
- **データベース**: Supabase
- **CDN**: Cloudflare

## 🔒 セキュリティとモデレーション

### 投稿品質管理
1. **3段階チェックシステム**
   - Tier 1: AI自動チェック（リアルタイム）
   - Tier 2: ユーザー通報
   - Tier 3: 管理者レビュー

2. **スパム対策**
   - 同一ユーザーの連続投稿制限（1日5投稿まで）
   - 同じ内容の重複投稿検出
   - 外部リンクの制限

3. **信頼性スコアリング**
   - ユーザーの投稿が通報されずにいいねを獲得 → スコア上昇
   - 通報された投稿が多い → スコア低下
   - 新規ユーザーは初期スコアから開始

### データプライバシー
- GDPR、個人情報保護法に準拠
- ユーザーデータの暗号化
- 退会時のデータ完全削除オプション

## 📊 成功指標 (KPI)

### フェーズ1（ローンチ後3ヶ月）
- 登録ユーザー数: 1,000人
- 投稿スポット数: 500件
- 月間アクティブユーザー (MAU): 500人
- 1投稿あたりの平均いいね数: 10

### フェーズ2（ローンチ後6ヶ月）
- 登録ユーザー数: 5,000人
- 投稿スポット数: 2,000件
- 月間アクティブユーザー (MAU): 2,000人
- ユーザー投稿率: 20%（ユーザーの20%が1件以上投稿）

### 定性的目標
- ユーザー満足度: 4.0/5.0以上
- 不適切コンテンツ率: 1%未満
- 投稿からいいねまでの平均時間: 24時間以内

## 💡 差別化ポイント

1. **リアルタイム性**
   - ユーザー投稿による「今」の情報
   - トレンド検出アルゴリズム

2. **コミュニティドリブン**
   - 実際に行った人のリアルな声
   - いいね数で信頼性を可視化

   - すぐに答えを見つけられる検索・フィルター

4. **安全性**

   - 健全なコミュニティ運営

## 📝 備考

### 開発優先順位
1. コア機能（投稿・閲覧・リアクション）
2. ユーザー認証
3. コンテンツモデレーション
4. 検索・フィルター機能
5. その他の機能

### リスクと対策
- **リスク**: 初期ユーザー獲得の難しさ
  - **対策**: クローズドコミュニティでのテストマーケティング、SNS活用

- **リスク**: 不適切コンテンツの流入
  - **対策**: 3層モデレーションシステム、コミュニティガイドライン明文化

- **リスク**: 情報の鮮度低下
  - **対策**: トレンドアルゴリズム、定期的な情報更新促進施策
